# 用例建模概念

  1. # 文本描述

  1. # 用例图

  1. # 什么是用例？

一个用例可以定义系统的一系列行为，通过此可为参与者提供有价值且可观测的结果。

  • 定义一个参与者要用到的系统功能

  • 描述系统为实现参与者价值所开展的行为序列

  • 对参与者与系统之间的交互活动进行建模

  • 从特定的用户角度出发，是完整的，实现特定用户价值的事件流

  1. # 参与者的定义

  • 与系统交互的人 或 与系统交互的硬件组件 或 其他的外部系统

  • 关注的重点是所承担的 “角色”

  • 命名明确且唯一
